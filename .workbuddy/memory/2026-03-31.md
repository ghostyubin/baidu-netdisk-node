两个 Dockerfile（srv/Dockerfile、xth/Dockerfile）将 `--frozen-lockfile` 改为 `--no-frozen-lockfile`，避免跨平台/CI 环境下 lockfile 不一致导致构建失败。

## 修复 workbench.vue 模板 TypeScript 语法错误

`packages/web/src/pages/workbench.vue` 第 61-62 行的 `@mouseenter`/`@mouseleave` 事件绑定中使用了 `(e as MouseEvent)` TypeScript 类型转换语法。Vue 模板编译器不支持 TS 语法，导致 Vite build 报错。已去掉类型断言，改为纯 JS 写法。

## 修复容器启动报错：baidu-netdisk-api/index.js not found

- `api`/`sdk` 包构建输出到各自 `dist/`，包根目录只有 `.ts` 源文件没有 `.js`
- `pnpm deploy` 按 `package.json` 的 `files` 字段打包，但 `index.js` 不存在
- 修复：`packages/srv/Dockerfile` 中先 `pnpm build api sdk`，再 `cp -r dist/. ./` 把产物拷回包根目录，最后再 `pnpm build srv web` 和 `pnpm deploy`

## 修复 sdk const enum 导致 Vite build 报错

`packages/sdk/src/types/enums.ts` 中所有枚举均为 `const enum`，编译后 JS 产物中不存在导出对象。Vite 构建 web 包时读取编译后的 `sdk/types.js`，找不到 `EUploadSteps`/`EDownloadSteps` 导出而报错。已将所有 `const enum` 改为普通 `enum`。

**具体改动**：
- `app.css`：`* {}` 去掉 `overflow: hidden`；`html/body/#app` 加 `overflow: hidden` 防页面整体滚动；新增 `.wb-main-area { overflow: hidden }`，移动端 `≤600px` 覆盖为 `overflow: visible`
- `workbench.vue`：Main area `<div>` 内联 overflow:hidden → 改为 class `wb-main-area`

## 移动端点击失效终极修复（2026-03-31，第三次）

**真正根本原因（第三层）**：全局 `* { flex-shrink: 0 }` 让所有 flex 子元素不收缩，导致 `.wb-main-content`（`overflow-y: auto`）虽然有滚动设置，但子元素无限向下撑，main 自身高度等于内容高度，没有形成真正的滚动容器 — 内容超出屏幕但无法点击。

**修复内容**：
- `app.css`：删除 `* { flex-shrink: 0 }`（保留 box-sizing/min-width/min-height）
- `app.css`：`.wb-sidebar` 补回 `flex-shrink: 0`（固定侧边栏宽度用）
- `app.css`：`.wb-bottom-nav-item` 改为 `flex: 1; flex-shrink: 1`（允许等分收缩）
- `app.css`：移动端 `.wb-main-content` 增加 `height: 0; flex: 1 1 0 !important` — 这是让 `overflow-y: auto` 真正生效的关键：有限高度才能触发滚动
- `app.css`：移动端 `.wb-main-area` 显式设为 `display: flex; flex-direction: column; height: 100%`
- `sync.vue`：移动端 `.sync-page` 去掉 `min-height: 100%`，改为纯 `height: auto; overflow: visible`，由上层 main 负责滚动
