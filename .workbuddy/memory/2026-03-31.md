# 2026-03-31

## GitHub Actions 改为推送到 ghcr.io

将两个 Docker 发布工作流（publish-srv-docker.yml、publish-xth-docker.yml）从 DockerHub 迁移到 ghcr.io：
- 鉴权改为使用内置 `secrets.GITHUB_TOKEN`，job 添加 `packages: write` 权限
- registry 改为 `ghcr.io`，镜像名为 `ghcr.io/${{ github.repository_owner }}/baidu-netdisk-sync` 和 `...auth`
- 新增 GHA 缓存层加速构建
- 新增 push tag 自动触发（srv-v* / xth-v*）

## Dockerfile 修复：pnpm install --no-frozen-lockfile

两个 Dockerfile（srv/Dockerfile、xth/Dockerfile）将 `--frozen-lockfile` 改为 `--no-frozen-lockfile`，避免跨平台/CI 环境下 lockfile 不一致导致构建失败。

## 修复 workbench.vue 模板 TypeScript 语法错误

`packages/web/src/pages/workbench.vue` 第 61-62 行的 `@mouseenter`/`@mouseleave` 事件绑定中使用了 `(e as MouseEvent)` TypeScript 类型转换语法。Vue 模板编译器不支持 TS 语法，导致 Vite build 报错。已去掉类型断言，改为纯 JS 写法。

## 修复容器启动报错：baidu-netdisk-api/index.js not found

- `api`/`sdk` 包构建输出到各自 `dist/`，包根目录只有 `.ts` 源文件没有 `.js`
- `pnpm deploy` 按 `package.json` 的 `files` 字段打包，但 `index.js` 不存在
- 修复：`packages/srv/Dockerfile` 中先 `pnpm build api sdk`，再 `cp -r dist/. ./` 把产物拷回包根目录，最后再 `pnpm build srv web` 和 `pnpm deploy`

## 修复 sdk const enum 导致 Vite build 报错

`packages/sdk/src/types/enums.ts` 中所有枚举均为 `const enum`，编译后 JS 产物中不存在导出对象。Vite 构建 web 包时读取编译后的 `sdk/types.js`，找不到 `EUploadSteps`/`EDownloadSteps` 导出而报错。已将所有 `const enum` 改为普通 `enum`。

- `workbench.vue` 用户卡片：无头像时改为渐变圆+首字母，而非单色 SVG

## workbench.vue：修复"未登录"显示 + 底部图标改版

**"未登录"修复**：`getUser()` 没有 `?id=xxx` 参数时，`users.find(item => item.id === undefined)` 永远找不到，回退到空用户显示"未登录"。修复为：无 id 时取第一个用户（`users[0]`），有 id 时才精确查找。

**底部图标改版**：
- 所有图标从 16px 放大到 18px

## widget-task.vue 新增速度显示

在任务卡片底部右侧新增实时速度显示：
- 通过 `watch(activeBytes)` 监听轮询数据变化，计算 diff/dt 得出速度
- 上传任务取 `upBytes`（秒传验证步骤取 `downBytes`），下载任务取 `downBytes`
- 速度值格式化为 B/s、KB/s、MB/s、GB/s
- bytes 无变化时保留上次速度（避免闪烁），任务停止/完成时清空
- 显示为带 accent 色的小 badge，右侧显示进度百分比
